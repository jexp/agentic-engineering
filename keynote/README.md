# Keynote — From Vibe Coding to Agentic Engineering

A ~30-minute conference keynote making the case that the practices which always worked with humans now matter more than ever — because AI agents are the newest kind of teammate, and they read the same code, docs, and tests we do.

## What this keynote argues

Three threads, woven together:

1. **The terminology shifted for a reason.** Karpathy retired "vibe coding" for serious work in early 2026. *"Agentic engineering"* names what's actually new — and what isn't.
2. **Good software design was never for the machine.** It was always for other humans who had to read, maintain, and extend the code. Now those humans include AI teammates trained on human-written code.
3. **The system around the model is the moat.** Models improve every six months. The practices, specs, tests, AGENTS.md, hooks, review discipline — those are yours.

The keynote is **inspiring & encouraging**, not dismissive of vibe coding (still useful for prototypes, learning, democratization) but honest about what production-grade work demands.
